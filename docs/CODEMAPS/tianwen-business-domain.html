1. Executive Summary

Tianwen is a Progressive Web App (PWA) restaurant POS system designed for small-to-medium dining operations in Taiwan. The app enables offline-first order management, employee time tracking, and cloud-based backup with real-time analytics.

Dimension Description
Value Proposition Offline-capable ordering; Cloudflare R2 cloud backup; multi-device sync; analytics dashboard with daily/product-level KPIs
Operational Modes Order entry (POS), clock-in / clock-out, attendance records, product & category config, analytics
Tech Foundation React 19 · SQLite WASM (OPFS) · Zustand · TanStack Query + Router · Vercel Serverless · Cloudflare R2
Persistence SQLite WASM in OPFS for offline; Vercel Functions as API; Cloudflare R2 for versioned backups
Device Support iPad 10 (2022) and later; portrait orientation enforced for touch usability
i18n Traditional Chinese (zh-TW) primary; English (en) secondary
User Roles Operator (order entry), Manager (employee/product admin), System Admin (backup/settings)
Backup Policy Manual + automatic (daily/weekly schedule); max 30 backups retained per device; gzip compression
Error Handling Multi-layer error boundaries, error overlay UI, SQLite-persisted error logs, user-facing recovery prompts
Auth Strategy Google OAuth for staff management binding only; no global user login required (device-based operation)

4. Business Rules & Invariants

The 13 rules below govern how the system behaves beyond the CRUD surface. Each rule lives in a specific place in code; edits that violate these rules should be treated as regressions.

4.1   Stall Category Auto-Memo Tagging
Any order containing commodities with type_id = "stall" automatically receives a memo tag 「攤位」. Marks orders as containing side items for kitchen display.
src/stores/order-store.ts:6–8 · submitOrder(memoTags?) line 82
4.2   Backup Scheduling with Taiwan Timezone Awareness
Backup overdue checks must respect Taiwan timezone (UTC+8), not server time. getTaiwanDayStart and getTaiwanWeekStart compute UTC milliseconds. On Tuesday TW 14:00, if last backup was Monday TW 23:00, daily schedule is NOT overdue.
src/lib/backup-schedule.ts · TAIWAN_OFFSET_MS
4.3   Backup Retention Policy (max 30 per device)
After successful upload, backend lists all .sqlite.gz, sorts by LastModified desc, and deletes oldest if count > 30. Automatic garbage collection ensures storage quota not exceeded.
api/backup/complete.ts:20 · MAX_BACKUPS = 30
4.4   Order Snapshot Price Protection
When an order is created, each item's price is captured into order_items.price. Subsequent price changes to commodities do NOT affect historical orders — revenue integrity and accounting are preserved.
src/stores/order-store.ts:63–69 · order-item-repository.ts
4.5   V1 → V2 Migration Idempotency
Legacy V1 data can be safely imported multiple times without duplication. INSERT OR IGNORE ensures interrupted imports can be resumed.
src/lib/v1-data-importer.ts:78 · buildInsertSql
4.6   Employee Termination Soft-Delete
Terminated employees are not removed from the DB. status set to "inactive" and resignation_date recorded. Historical orders/attendance preserved for audit; Clock-In UI filters to active only.
src/lib/schema.ts:92–93 · CHECK constraint
4.7   Order Number Sequence (customer-facing)
Each order receives an auto-incrementing number field independent of its nanoid id, used for receipt printing and customer reference. Computed via SELECT MAX(number)+1.
src/lib/repositories/order-repository.ts · getNextOrderNumber
4.8   Minimum Init UI Display Duration (5 seconds)
InitOverlay must display for at least 5 seconds even if DB is ready. Prevents jarring flash on fast machines; ensures user perceives app as "doing work".
src/stores/init-store.ts:66–73 · shownAt timestamp
4.9   Portrait-Only Orientation Enforcement
App locked to portrait on mobile/tablet. RootLayout detects landscape via matchMedia('(orientation: portrait)') and shows WaitingOverlay with rotate prompt.
src/routes/route-tree.tsx:23–41 · useSyncExternalStore
4.10   Device Identity Tracking
Each browser/device has a unique DEVICE_ID persisted to localStorage. Backup filenames include device code: tianwen-<device>-<date>.sqlite.gz — enables per-device restore and cross-device listing.
src/lib/device.ts · getDeviceId()
4.11   No Global User Login Required
App operates without a global user login for order entry. OrderPage loads directly — no operator selection step. Admin functions (staff mgmt, products, backup) are gated by AuthGuard + Google OAuth. The store has operatorId / operatorName fields but no UI sets them (planned feature).
src/components/auth-guard/auth-guard.tsx · AuthGuard
4.12   Order Edit Rewrites Items (Original Total Preserved)
Editing a submitted order deletes old order_items and order_discounts and inserts fresh rows. original_total is preserved for audit trail if edits change the total.
src/stores/order-store.ts · startEditOrder + submitOrder
4.13   Gzip Compression for Backup Transport
Database backup compressed via browser CompressionStream('gzip') before uploading to R2. Typical 5 MB SQLite → 1-2 MB compressed (60-80% reduction).
src/lib/backup.ts:26–83 · compress / decompress

State boundaries

  • Zustand — ephemeral client state (cart, active overlays, init progress, backup UI)
  • TanStack Query — server-state cache over DB reads; invalidation on writes
  • SQLite / OPFS — durable source of truth; lives in Web Worker to avoid UI blocking
  • localStorage — DEVICE_ID, v1 backup detection flag, first-run markers

Overlay system

  • Three overlays: InitOverlay (bootstrap / import), ErrorOverlay (error animation), WaitingOverlay (portrait rotate prompt)
  • Each uses React portal to document.body to escape ancestor containing blocks from transform / backdrop-filter
  • Registration via useRegisterOverlay hook (ref-counted in init-store.activeOverlays); AppHeader auto-switches to transparent + white-text styling while any overlay is active
